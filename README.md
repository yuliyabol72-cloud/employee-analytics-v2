# Аналитика результатов тестирования

Современное веб-приложение для бизнес-тренеров телекоммуникационной компании. Автоматический анализ результатов тестирования сотрудников, построение наглядной аналитики и формирование готовых выводов для руководителей.

## 📋 Описание проекта

Приложение предназначено для анализа результатов тестирования сотрудников после проведения обучения. Оно автоматически обрабатывает Excel-файлы с результатами, вычисляет статистику, выявляет проблемные темы и формирует рекомендации для улучшения обучения.

## ✨ Возможности

### 📊 Аналитика
- **Многоуровневый анализ**: по городам → подразделениям → компании
- **Статистика по сотрудникам**: количество, успешность, средний результат
- **Анализ по вопросам**: процент правильных ответов, индекс сложности
- **Анализ по темам**: объединение вопросов по темам обучения
- **Тепловая карта**: визуализация результатов по городам и вопросам

### 📈 Визуализация
- KPI-карточки с ключевыми показателями
- Графики распределения результатов
- Рейтинги городов и подразделений
- ТОП проблемных вопросов и тем
- Интерактивные графики Chart.js

### 🔍 Фильтрация
- Фильтрация по городу, подразделению, должности, дате
- Поиск по таблице сотрудников
- Автоматическое пересчет всех показателей

### 📋 Отчеты
- Автоматический аналитический отчет
- Рекомендации на основе данных
- Экспорт в PDF
- Экспорт в Excel
- Отчет для руководителя

### 👥 Детальный анализ
- Карточка сотрудника с результатами
- Список вопросов с ошибками
- Темы для повторения
- Сильные стороны и зоны развития

## 📁 Структура Excel файла

### Основной лист "Результаты"

Обязательные столбцы:
- `Город` - город сотрудника
- `Подразделение` - подразделение (например, ЦАП, ЦРК)
- `ФИО` - полное имя сотрудника
- `Должность` - должность
- `Дата тестирования` - дата прохождения теста
- `Общий процент` - общий процент прохождения теста

Столбцы вопросов (формат: `Вопрос N`):
- `Вопрос 1` - результат первого вопроса (1 - правильно, 0 - неправильно)
- `Вопрос 2` - результат второго вопроса
- ...
- `Вопрос N` - результат N-го вопроса

### Дополнительный лист "Справочник вопросов" (опционально)

Столбцы:
- `Номер вопроса` - номер вопроса
- `Тема обучения` - тема, к которой относится вопрос

Пример:
```
Номер вопроса | Тема обучения
1            | Приветствие
2            | Выявление потребностей
3            | Работа с возражениями
```

## 🎯 Проходной балл

Проходной балл составляет **80%**. Сотрудник считается успешно прошедшим тестирование при результате 80% и выше.

## 📊 KPI показатели

Приложение отображает следующие показатели:
- Всего сотрудников
- Успешно прошли
- Не прошли
- Процент успешной сдачи
- Процент несдавших
- Средний результат по компании
- Максимальный результат
- Минимальный результат
- Количество городов
- Количество подразделений
